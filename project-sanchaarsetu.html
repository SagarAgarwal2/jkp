Project Overview

Sanchaar Setu ("Bridge of Communication") is an enterprise middleware platform designed for Indian e-governance infrastructure. It enables secure, reliable, and auditable communication between independent government departments — solving the critical challenge of data consistency and traceability across disparate systems.

Architected for a government summit submission, it incorporates high-reliability features like Kafka Dead Letter Queues, exponential backoff retry, and tamper-evident hash-chained audit logs to meet production-grade compliance requirements.

  • Kafka-Based Messaging: Asynchronous, fault-tolerant message delivery between departments with DLQ and retry mechanisms
  • Hash-Chained Audit Logs: Tamper-evident logging using SHA-256 hash chains — any modification breaks the chain
  • PII Scrubbing: Automatic redaction of Aadhaar, PAN, phone, and email from all log outputs
  • RBAC Authorization: Role-based access control (Admin, Operator, Auditor) with JWT authentication
  • Rate Limiting: Redis-backed sliding window rate limiting to prevent API abuse
  • Two-Way Sync: Middleware propagates data changes and resolves conflicts between central portals and department systems

Security & Compliance

  • Tamper-Evident Audit Trail: Every log entry is cryptographically linked via SHA-256 hash chains. Any modification is instantly detectable.
  • PII Auto-Redaction: Regex-based scrubbing of sensitive data from all log output for data protection compliance.
  • JWT Authentication: Stateless token-based auth with configurable expiry and role claims.
  • Rate Limiting: Redis-backed sliding window algorithm protecting endpoints from abuse and DDoS.

Technical Implementation

  • FastAPI: High-performance async Python web framework for the API gateway
  • Apache Kafka: Distributed event streaming for reliable inter-department messaging
  • PostgreSQL: Persistent storage for audit logs, department data, and sync state
  • Redis: In-memory caching for rate limiting and session management
  • Docker Compose: Multi-container orchestration for all services
  • Exponential Backoff: Intelligent retry logic with configurable max retries
  • Dead Letter Queue: Failed messages routed to DLQ for manual review and reprocessing
